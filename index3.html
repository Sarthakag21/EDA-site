Exploratory Data Analysis (EDA)

From History to Tools, Techniques and Practical Applications

1. Introduction

Exploratory data analysis or "EDA" is a critical first step in analyzing the data. It is used by data scientists to analyze and investigate data sets and summarize their main characteristics, often employing data visualization methods.

Main reasons we use EDA:

  • detection of mistakes
  • checking of assumptions
  • preliminary selection of appropriate models
  • determining relationships among the explanatory variables

2. History of EDA

EDA was primarily developed by John Tukey starting in the 1970s as:

  • procedures for analyzing data
  • techniques for interpreting the results of such procedures
  • ways of planning the gathering of data to make its analysis easier
  • more precise or more accurate

3. Steps Involved in EDA

  1. Data Collection

    2. Collect the required data from various sources such as databases, web scraping, or APIs.

  2. Data Cleaning
    • Handle missing values
    • Remove duplicates
    • Correct data types
  3. Data Transformation
    • Normalize or standardize the data
    • Aggregate or disaggregate data based on analysis
  4. Data Exploration
    • Univariate Analysis: Analyze individual variables using summary statistics and visualizations (e.g., histograms, box plots).
    • Bivariate Analysis: Analyze the relationship between two variables with scatter plots, correlation coefficients, and cross-tabulations.
    • Multivariate Analysis: Investigate interactions between multiple variables using pair plots and correlation matrices.
  5. Data Visualization

    6. Visualize data distributions and relationships using visual tools such as bar charts, line charts, scatter plots, heatmaps, and box plots.

  6. Identify Patterns and Outliers

    7. Detect patterns, trends, and outliers in the data using visualizations and statistical methods.

  7. Data Summarization

    8. Summarize findings with descriptive statistics, visualizations, and key insights.

4. Why EDA Matters (Use Cases)

  • Understanding distribution and spread of data
  • Identifying missing data and outliers
  • Uncovering hidden patterns and trends
  • Preprocessing decisions before machine learning

Example: In healthcare, EDA is used to study patient data to detect health trends, identify risk factors, and improve decision-making.

5. Key Features of EDA

  • Descriptive statistics
  • Data visualization
  • Outlier detection
  • Correlation analysis

6. Types of EDA

6.1 Univariate Analysis

  • Focuses on analyzing a single variable at a time.
  • To understand the variable's distribution, central tendency, and spread.
  • Techniques used: mean, median, mode, histograms, box plots, etc.

6.2 Bivariate Analysis

  • Examines the relationship between two variables.
  • To understand how one variable affects or is associated with another.
  • Techniques used: scatter plots, line plots, pair plots, correlation coefficients, etc.

6.3 Multivariate Analysis

  • Analyzes more than two variables simultaneously.
  • Helps to uncover complex interactions and relationships among variables.
  • Techniques used: multiple regression, PCA, factor analysis, clustering, etc.

6.4 Dimensionality Reduction

  • Reduces the number of input variables in a dataset.
  • Helps to remove noise and improve model performance.
  • Techniques used: PCA (Principal Component Analysis), t-SNE, LDA, etc.

8. Techniques Used in EDA

8.1 Box Plot

  • Box plots also known as box-and-whisker plots
  • Standardized way of displaying the distribution of data based on a five-number summary: minimum, first quartile (Q1), median, third quartile (Q3), and maximum.

Key Features:

  • Box: Represents the interquartile range (IQR), which is the range between Q1 and Q3.
  • Median: A vertical line within the box indicating the middle value of the data.
  • Whiskers: Extend from the box to the minimum and maximum values within a certain range.
  • Outliers: Data points that fall outside the whiskers, often displayed as individual points.

8.2 Histogram

  • It is a graphical representation of the distribution of numerical data.
  • It uses bars to show the frequency of data points within specified intervals or bins.

Key Features:

  • Bins: Data is grouped into ranges called bins or intervals.
  • Bars: Each bin is represented by a bar, with the height of the bar indicating the frequency.
  • Frequency Density: The height of the bar can represent either the frequency (count) or the frequency density (frequency divided by the bin width).

8.3 Scatter Plot

  • It is a type of graph that displays values for typically two variables as a set of points on a two-dimensional plane.
  • It's used to visualize the relationship or correlation between these variables.

Key Features:

  • Cartesian Coordinates: The plot uses a standard x-y coordinate system, where each axis represents one of the variables.
  • Correlation: Scatter plots can help identify whether there is a relationship between the variables and the nature of that relationship (positive, negative, or no correlation).

8.4 Heat Map

  • It is a graphical representation of data where values are depicted by color.
  • It uses a system of color coding to represent different values, making it easy to visualize complex information and identify patterns or trends.
  • Commonly used in various fields, including web analytics, user experience (UX) design, and data visualization.

Key Features:

  • Color-Coded Data Representation: It uses color intensity to represent different data values. Warmer colors usually indicate higher values, while cooler colors represent lower values.
  • Grid or Matrix Format: Data is organized in a grid or matrix format, with each cell representing a specific data point. The color of each cell corresponds to the value of that data point.

8.5 Bar Chart

  • It is a chart that uses rectangular bars to display and compare data across different categories or groups.
  • The length or height of each bar is proportional to the value it represents.

Key Features:

  • Rectangular bars: Data is represented by a series of rectangular bars.
  • Categorical data: Bar charts are typically used to display data that can be categorized into distinct groups.

9. Real-World EDA Example: Titanic Dataset

The Titanic dataset is a classic for practicing EDA. It contains information about passengers on the Titanic, including their age, class, gender, and survival status.

        
          import seaborn as sns
          import pandas as pd
          import matplotlib.pyplot as plt

          df = sns.load_dataset('titanic')
          print(df.head())
          sns.countplot(x='survived', data=df)
          plt.title('Survival Count')
          plt.show()

This basic plot shows how many people survived vs. died on the Titanic.

10. Common EDA Mistakes to Avoid

  • Ignoring missing values or assuming they are random
  • Overlooking data types and categorical encodings
  • Failing to explore variable relationships
  • Using inappropriate visualizations for the data type

11. Conclusion

EDA is the foundation of any successful data project. It ensures that data scientists deeply understand their data before building predictive models. Visualizations and statistical summaries make patterns and problems in the data transparent and easy to interpret.
